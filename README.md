## Overview

Segment Studio lets you upload a CSV file, run automatic K-Means clustering, and receive human-friendly names and descriptions for each cluster — generated by the `meta-llama/Meta-Llama-3-8B-Instruct` model through the Hugging Face Inference API.

**Key features:**

- CSV upload with instant data preview
- WCSS (Elbow method) calculation to help choose the optimal number of clusters
- K-Means clustering with manual k selection
- Automatic cluster naming and description via LLaMA
- Export results as a CSV with an added `cluster_name` column

---

## Requirements

- Python `3.9` or higher
- Internet connection (required for the Hugging Face Inference API)
- A Hugging Face access token with access to `meta-llama/Meta-Llama-3-8B-Instruct`

## API Key Setup

Replace the placeholder value inside `get_HF_client()` in the source code:

```python
hf_token = "YOUR_HF_API_KEY_HERE"
```

**Recommended:** Store the token as an environment variable instead of hardcoding it:

```bash
export HF_TOKEN="hf_xxxxxxxxxxxxxxxxxxxx"
```

Then update the code to read from the environment:

```python
import os
hf_token = os.environ.get("HF_TOKEN")
```

To get an access token: [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)

> ⚠️ **Warning:** Never commit API keys directly in code pushed to GitHub. Use environment variables or a `.env` file (with a proper `.gitignore`).

## Usage Workflow

### Step 1 — Upload CSV
Upload a CSV file containing the data you want to cluster. A preview table will be displayed.

### Step 2 — WCSS (Elbow Plot)
Select a range of k values and click **Run WCSS**. Use the Elbow plot to identify the optimal number of clusters.

### Step 3 — Create Clusters
Choose your desired k value and click **Create clusters**. A table showing the row count per cluster will appear.

### Step 4 — Generate Names & Descriptions (LLaMA)
Click **Generate names/descriptions with LLaMA**. The model receives a statistical summary of each cluster and returns a short name and a one-line description.

### Step 5 — Export
Download the clustered CSV with the `cluster_name` column appended to the original data.

## Code Structure

| Function | Description |
|---|---|
| `wcss_by_k(X, k_min, k_max)` | Runs K-Means for each k in range and returns a DataFrame with WCSS values |
| `fit_kmeans_labels(X, k)` | Fits K-Means and returns cluster label assignments for each row |
| `get_HF_client()` | Initializes the Hugging Face Inference Client (cached with `@st.cache_resource`) |
| `ask_llama_for_name_desc(summary)` | Sends a cluster summary to LLaMA and returns a name + description tuple |
| `preprocess(df_features)` | Scales numeric columns and one-hot encodes categorical columns |
| `build_cluster_summary(df, labels)` | Builds a text summary of each cluster for use as an LLM prompt |

---

## Notes & Limitations

- `Meta-Llama-3-8B-Instruct` is a gated model — you must request access on Hugging Face before use.
- The free Inference API is subject to rate limits. For heavy usage, consider a dedicated Inference Endpoint.
- Only statistical summaries (means, top values) are sent to the API — no raw rows are transmitted.
- The exported CSV preserves all original columns and adds a single `cluster_name` column.
